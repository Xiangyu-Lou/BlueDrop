# 聚音 (BlueDrop) - 产品需求文档 (PRD)

> **版本：** 1.0
> **日期：** 2026-03-24
> **阶段：** Phase 1 — iOS → Windows 单向音频传输

---

## 1. 产品定位

聚音 (BlueDrop) 是一款音频路由与混流桌面软件，核心目标是让用户将 iPhone 的媒体音频通过蓝牙 A2DP 传输到 Windows PC，与本地麦克风混合后输出至虚拟麦克风，供 Discord、腾讯会议、Zoom 等通讯软件调用。

**Phase 1 范围：** iPhone (iOS) → Windows (10 2004+ / 11) 单向传输。

---

## 2. 目标用户与使用场景

| 场景 | 描述 |
|------|------|
| **在线会议分享手机音频** | 用户在 Zoom/Teams 会议中，需要将手机上播放的音频（录音、音乐片段）实时共享给其他参会者，同时自己仍可通过麦克风讲话 |
| **直播/录制混音** | 主播需要将手机端的伴奏/音效与自己的语音混合后推送至 OBS 等直播软件 |
| **语音通话协同** | 用户在 PC 端进行语音通话时，需要将手机来电/语音消息的声音转发给对方 |

---

## 3. 运行环境与前置条件

### 3.1 系统要求

| 项目 | 要求 |
|------|------|
| **操作系统** | Windows 10 Version 2004 (Build 19041) 及以上，或 Windows 11 |
| **架构** | 仅 x64 |
| **蓝牙** | 支持 Bluetooth 4.0+ 的蓝牙适配器（内置或外置均可） |
| **发送端** | iPhone (iOS 13+)，原生支持 A2DP |

### 3.2 外部依赖

| 依赖 | 说明 |
|------|------|
| **VB-Audio Virtual Cable** | 用户需自行安装。软件启动时检测其存在，若未检测到则给出安装引导提示 |

### 3.3 启动校验（强制）

软件启动时执行以下校验，任一不通过则阻断运行并提示：

1. **OS 版本校验** — 低于 Win10 Build 19041 → 提示升级系统
2. **蓝牙适配器校验** — 未检测到蓝牙硬件 → 提示插入蓝牙适配器
3. **虚拟声卡校验** — 未检测到 VB-Cable 设备 → 提示安装并提供下载链接

## 4. 功能需求

### F1: 蓝牙 A2DP Sink 接收

| 属性 | 描述 |
|------|------|
| **功能** | 使 PC 作为 A2DP Sink 暴露给 iPhone，接受配对并接收立体声媒体音频 |
| **采样率** | 44.1kHz / 48kHz（自适应） |
| **声道** | 立体声 (Stereo) |
| **编解码** | SBC（必须），AAC（可选，优先） |
| **状态管理** | 未连接 → 等待配对 → 已连接（显示设备名称）→ 断开 |
| **参考实现** | [AudioPlaybackConnector](https://github.com/ysc3839/AudioPlaybackConnector)（Win10 2004+ A2DP Sink） |

### F2: 音频设备枚举与捕获

| 属性 | 描述 |
|------|------|
| **输入设备枚举** | 列出系统所有活跃的音频输入（录音）设备 |
| **输出设备枚举** | 列出系统所有活跃的音频输出（播放）设备 |
| **虚拟设备识别** | 自动识别 VB-Cable 输入端（CABLE Input），在列表中高亮标记 |
| **热插拔响应** | 设备插入/拔出时实时刷新列表，并在 UI 中更新 |
| **麦克风捕获** | 通过 WASAPI 实时捕获用户指定的物理麦克风音频流 |

### F3: 双路由音频引擎

软件内部并行运行两条独立的音频路由：

```
┌─────────────┐
│  iPhone BT   │──┬──→ 路由 A: 直出物理耳机（本地监听）
│  (A2DP Sink) │  │
└─────────────┘  │    ┌─────────────┐
                  ├──→ │   混音器     │──→ 路由 B: 输出至 VB-Cable Input
┌─────────────┐  │    └─────────────┘     （供通讯软件读取）
│  物理麦克风   │──┘
└─────────────┘
```

| 路由 | 输入 | 输出 | 说明 |
|------|------|------|------|
| **A — 本地监听** | 蓝牙手机音频 | 用户指定的物理耳机 | 不含麦克风声音，避免回音 |
| **B — 通讯广播** | 蓝牙手机音频 + 物理麦克风 | VB-Cable Input | 混合后供第三方软件读取 |

### F4: 音量与通道控制

| 控制项 | 类型 | 说明 |
|--------|------|------|
| **手机音频增益** | 滑块 (0% ~ 200%) | 调节蓝牙接收音频的输入音量 |
| **麦克风增益** | 滑块 (0% ~ 200%) | 调节物理麦克风的输入音量 |
| **手机混入比例** | 滑块 (0% ~ 100%) | 控制手机音频在路由 B 中的混合比例 |
| **手机音频静音** | 开关 | 静音手机音频（不中断底层数据流） |
| **麦克风静音** | 开关 | 静音麦克风（不中断底层数据流） |

```
│  ● 设置   │  │ 蓝牙状态 / 设备名称          │  │
│          │  └──────────────────────────────┘  │
│          │                                  │
│          │  ┌─ 设备选择卡片 ──────────────┐  │
│          │  │ 麦克风 / 耳机 / 虚拟声卡     │  │
│          │  └──────────────────────────────┘  │
│          │                                  │
│          │  ┌─ 混音台卡片 ────────────────┐  │
│          │  │ 手机音量 ████████░░ 🔇      │  │
│          │  │ 麦克风   ██████░░░░ 🔇      │  │
│          │  └──────────────────────────────┘  │
│          │                                  │
│  版本信息  │                                  │
└──────────┴──────────────────────────────────┘
```

### 5.2 页面定义

| 页面 | 内容 |
|------|------|
| **主页** | 连接状态卡片 + 设备选择卡片 + 混音台卡片（核心操作集中于一页） |
| **设置** | OS 信息展示、VB-Cable 状态检测、蓝牙适配器信息、关于/版本号 |

### 5.3 连接状态卡片

| 状态 | 显示 |
|------|------|
| 未连接 | 灰色图标 + "未连接蓝牙设备" |
| 等待配对 | 蓝色脉冲动画 + "等待 iPhone 连接..." |
| 已连接 | 绿色图标 + "已连接：[设备名称]" + 音频格式信息 (如 48kHz/Stereo/AAC) |
| 连接断开 | 橙色图标 + "连接已断开，等待重新连接..." |

### 5.4 设备选择卡片

三个下拉菜单：

1. **物理麦克风** — 列出所有音频输入设备，默认选择系统默认录音设备
2. **本地监听耳机** — 列出所有音频输出设备，默认选择系统默认播放设备
3. **虚拟麦克风输出** — 列出所有音频输入设备中的虚拟设备，自动高亮 CABLE Input

### 5.5 混音台卡片

- 手机音频：水平音量推子 + 数值显示 + 静音按钮
- 麦克风：水平音量推子 + 数值显示 + 静音按钮
- 手机混入比例：水平推子（仅影响路由 B）

## 6. 非功能性需求

### 6.1 性能

| 指标 | 要求 |
|------|------|
| **内部处理延迟** | 从蓝牙接收到虚拟麦克风输出 ≤ 50ms |
| **音频引擎** | 基于 WASAPI 共享模式或独占模式 |
| **CPU 占用** | 空闲状态 < 1%，工作状态 < 5% |
| **内存占用** | < 100MB |

### 6.2 稳定性与异常处理

| 场景 | 行为 |
|------|------|
| **蓝牙意外断开** | 停止推流 → UI 显示"连接断开" → 自动监听重连 → 恢复后自动恢复音频流 |
| **物理设备拔出** | 停止相关路由 → UI 提示设备已断开 → 刷新设备列表 → 设备重新插入后允许手动恢复 |
| **VB-Cable 未安装** | 启动时阻断并提示 → 运行中被卸载则停止路由 B 并提示 |
| **禁止崩溃** | 所有异常必须被捕获并优雅处理，任何情况下不允许未处理异常导致进程终止 |

### 6.3 兼容性

| 项目 | 要求 |
|------|------|
| **最低系统** | Windows 10 Version 2004 (Build 19041) x64 |
| **推荐系统** | Windows 11 22H2+ x64 |
| **蓝牙适配器** | 任何支持 Bluetooth 4.0+ 的适配器（需系统正确安装驱动） |
| **DPI 缩放** | 支持 100%/125%/150%/200% 系统缩放 |

## 8. 验收标准 (Phase 1 MVP)

- [ ] iPhone 可通过蓝牙发现并连接 PC，PC 以 A2DP Sink 身份接收立体声音频
- [ ] 用户可选择物理麦克风、监听耳机、虚拟声卡输出
- [ ] 路由 A：手机音频可从指定耳机播出，无麦克风回音
- [ ] 路由 B：手机音频 + 麦克风混合后可在 VB-Cable Input 端被第三方软件读取
- [ ] 各音量推子和静音开关实时生效
- [ ] 蓝牙断开后 UI 正确提示，重连后自动恢复
- [ ] 内部处理延迟 ≤ 50ms
- [ ] 支持系统 DPI 缩放 (100%~200%)
